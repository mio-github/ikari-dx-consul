# イカリ消毒様 DXコンサルティングPoC 必要ドキュメント一覧

**プロジェクト期間**: 2025年11月〜2026年1月末（3ヶ月間）

## 📋 ドキュメント体系

### 1. プロジェクト管理ドキュメント（事前準備フェーズ：〜11月上旬）

#### 1.1 NDA（機密保持契約書）
- **目的**: 情報の機密保持
- **内容**: 双方の機密情報保護条項
- **提出時期**: プロジェクト開始前
- **形式**: PDF

#### 1.2 ヒアリング計画表
- **目的**: ヒアリングの効率的な実施
- **内容**:
  - ヒアリング対象部署一覧
  - 日程調整表
  - 質問項目リスト
  - 必要資料リスト
- **提出時期**: 11月上旬
- **形式**: Excel / Googleスプレッドシート

---

### 2. 現状把握フェーズドキュメント（PoCフェーズ：11月中旬）

#### 2.1 現行業務フロー図
- **目的**: 各部署の業務プロセスの可視化
- **内容**:
  - 部署別業務フロー
  - データの流れ
  - システム間連携
  - 業務の入出力
- **提出時期**: 11月下旬
- **形式**: PowerPoint / Miro / draw.io

#### 2.2 課題リスト
- **目的**: 現状の課題を体系的に整理
- **内容**:
  - 非効率な業務の特定
  - 属人化リスク
  - 情報断絶ポイント
  - システム課題
  - 優先度評価
- **提出時期**: 11月下旬
- **形式**: Excel / Notion

#### 2.3 As-Isマップ（現状業務マップ）
- **目的**: 現状業務・データの全体像を可視化
- **内容**:
  - 全体業務フロー
  - データフロー図
  - システム構成図
  - 部門間連携図
- **提出時期**: 12月初旬
- **形式**: PowerPoint / Miro

#### 2.4 システム連携図
- **目的**: 組織全体のシステム構成と連携を把握
- **内容**:
  - 使用中システム一覧
  - システム間データ連携
  - クラウド利用状況
  - ライセンス情報
  - サーバ構成
- **提出時期**: 12月初旬
- **形式**: Visio / draw.io / PowerPoint

## 🎯 成果物の特徴

### ✅ 3つの重要ポイント

1. **可視化重視**
   - すべての成果物はビジュアル化
   - 経営層・現場双方が理解しやすい形式
   - As-Is/To-Beの対比を明確化

2. **実行可能性**
   - 具体的なアクションプラン
   - 担当者・責任者の明確化
   - 実現可能な費用見積もり

3. **段階的アプローチ**
   - 短期・中期・長期の明確な区分
   - 優先順位付けと依存関係の整理
   - クイックウィン施策の特定
